# Privacy & Data Sanitation

This repository is intentionally free of personal information and secrets. The project documentation is written in a neutral tone and focuses solely on the PRYMA system.

## Current state
- No personal identifiers, credentials, or private contact details are stored in this repository.
- Content is limited to architectural concepts, protocols, and design outlines for PRYMA.

## Contribution guidelines
- Do not commit logs, transcripts, or notes that reference individuals or private communications.
- Avoid adding credentials, API keys, or configuration values tied to personal accounts.
- Prefer generalized examples over real-world data when illustrating usage or behavior.

If sensitive information is discovered, remove it immediately and rotate any affected credentials.
